## Overview

NEAR Connect provides comprehensive support for signing and sending transactions. This guide covers everything from simple transfers to complex multi-action transactions.

## Basic Transaction

The simplest transaction is a NEAR token transfer:

```typescript
import { NearConnector } from "@hot-labs/near-connect";

const connector = new NearConnector();
const wallet = await connector.wallet();

const result = await wallet.signAndSendTransaction({
  receiverId: "recipient.near",
  actions: [{
    type: "Transfer",
    params: {
      deposit: "1000000000000000000000000", // 1 NEAR in yoctoNEAR
    },
  }],
});

console.log(`Transaction hash: ${result.transaction.hash}`);
```

## Action Types

NEAR Connect supports all NEAR action types. You can use two formats:

1. **ConnectorAction format** (recommended for simplicity)
2. **near-api-js Action format** (for advanced use cases)

### Transfer

Send NEAR tokens to another account:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "recipient.near",
  actions: [{
    type: "Transfer",
    params: {
      deposit: "1000000000000000000000000", // 1 NEAR
    },
  }],
});
```

### Function Call

Call a smart contract method:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "contract.near",
  actions: [{
    type: "FunctionCall",
    params: {
      methodName: "set_greeting",
      args: { greeting: "Hello, NEAR!" },
      gas: "30000000000000", // 30 TGas
      deposit: "0", // No NEAR attached
    },
  }],
});
```

<Note>
`args` can be a JSON object or a Uint8Array. When using an object, it will be automatically JSON-stringified.
</Note>

### Create Account

Create a new NEAR account:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "newaccount.near",
  actions: [{
    type: "CreateAccount",
  }],
});
```

### Delete Account

Delete an account and transfer remaining balance:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "account-to-delete.near",
  actions: [{
    type: "DeleteAccount",
    params: {
      beneficiaryId: "beneficiary.near",
    },
  }],
});
```

### Add Key

Add a full access or function call access key:

```typescript Full Access Key
await wallet.signAndSendTransaction({

    type: "AddKey",
    params: {
      publicKey: "ed25519:...",
      accessKey: {
        nonce: 0,
        permission: "FullAccess",
      },
    },
  }],

```

```typescript Function Call Key
await wallet.signAndSendTransaction({
  receiverId: "account.near",
  actions: [{
    type: "AddKey",
    params: {
      publicKey: "ed25519:...",
      accessKey: {
        nonce: 0,
        permission: {
          receiverId: "contract.near",
          methodNames: ["method1", "method2"],
          allowance: "250000000000000000000000", // 0.25 NEAR
        },
      },
    },
  }],
});
```

### Deploy Contract

Deploy a smart contract:

```typescript
const wasmCode = await fetch("contract.wasm").then(r => r.arrayBuffer());

await wallet.signAndSendTransaction({
  receiverId: "contract.near",
  actions: [{
    type: "DeployContract",
    params: {
      code: new Uint8Array(wasmCode),
    },
  }],
});
```

### Stake

Stake NEAR tokens with a validator:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "validator.near",
  actions: [{
    type: "Stake",
    params: {
      stake: "100000000000000000000000000", // 100 NEAR
      publicKey: "ed25519:...",
    },
  }],
});
```

## Multiple Actions

You can include multiple actions in a single transaction:

```typescript
await wallet.signAndSendTransaction({
  receiverId: "contract.near",
  actions: [
    {
      type: "FunctionCall",
      params: {
        methodName: "deposit",
        args: {},
        gas: "30000000000000",
        deposit: "1000000000000000000000000", // 1 NEAR
      },
    },
    {
      type: "FunctionCall",
      params: {
        methodName: "stake",
        args: { amount: "1000000000000000000000000" },
        gas: "30000000000000",
        deposit: "0",
      },
    },
  ],
});
```

<Tip>
All actions in a single transaction are atomic - they all succeed or all fail together.
</Tip>

## Multiple Transactions

Send multiple transactions that will be processed sequentially:

```typescript
const results = await wallet.signAndSendTransactions({
  transactions: [
    {
      receiverId: "contract1.near",
      actions: [{
        type: "FunctionCall",
        params: {
          methodName: "method1",
          args: {},
          gas: "30000000000000",
          deposit: "0",
        },
      }],
    },
    {
      receiverId: "contract2.near",
      actions: [{
        type: "FunctionCall",
        params: {
          methodName: "method2",
          args: {},
          gas: "30000000000000",
          deposit: "0",
        },
      }],
    },
  ],
});

results.forEach((result, index) => {
  console.log(`Transaction ${index} hash: ${result.transaction.hash}`);
});
```

<Warning>
If any transaction in the batch fails, subsequent transactions will not be processed.
</Warning>

## Specifying Signer

By default, transactions are signed by the first connected account. You can specify a different signer:

```typescript
const accounts = await wallet.getAccounts();

await wallet.signAndSendTransaction({
  signerId: accounts[1].accountId, // Use second account
  receiverId: "recipient.near",
  actions: [{
    type: "Transfer",
    params: { deposit: "1000000000000000000000000" },
  }],
});
```

## Network Override

Override the default network for a specific transaction:

```typescript
await wallet.signAndSendTransaction({
  network: "testnet",
  receiverId: "recipient.testnet",
  actions: [{
    type: "Transfer",
    params: { deposit: "1000000000000000000000000" },
  }],
});
```

## Handling Transaction Results

The transaction result contains detailed information:

```typescript
const result = await wallet.signAndSendTransaction({
  receiverId: "contract.near",
  actions: [{
    type: "FunctionCall",
    params: {
      methodName: "get_number",
      args: {},
      gas: "30000000000000",
      deposit: "0",
    },
  }],
});

// Transaction details
console.log(`Hash: ${result.transaction.hash}`);
console.log(`Signer: ${result.transaction.signer_id}`);
console.log(`Receiver: ${result.transaction.receiver_id}`);

// Receipt outcomes
result.receipts_outcome.forEach((receipt) => {
  console.log(`Receipt ID: ${receipt.id}`);
  console.log(`Gas burnt: ${receipt.outcome.gas_burnt}`);
  
  // Logs from the contract
  receipt.outcome.logs.forEach((log) => {
    console.log(`Log: ${log}`);
  });
  
  // Return value (if any)
  if (receipt.outcome.status.SuccessValue) {
    const value = JSON.parse(
      Buffer.from(receipt.outcome.status.SuccessValue, "base64").toString()
    );
    console.log(`Return value: ${value}`);
  }
});

// Transaction status
if (result.status.SuccessValue !== undefined) {
  console.log("Transaction succeeded");
} else if (result.status.Failure) {
  console.error("Transaction failed:", result.status.Failure);
}
```

## Using near-api-js Actions

For advanced use cases, you can use actions from `near-api-js`:

```typescript
import { transactions } from "near-api-js";

await wallet.signAndSendTransaction({
  receiverId: "contract.near",
  actions: [
    transactions.functionCall(
      "method_name",
      { arg1: "value" },
      BigInt(30_000_000_000_000), // gas
      BigInt(0) // deposit
    ),
  ],
});
```

<Note>
NEAR Connect automatically detects and handles both action formats.
</Note>

## Delegate Actions

Sign delegate actions for meta transactions:

```typescript
const accounts = await wallet.getAccounts();
const senderId = accounts[0].accountId;

const { signedDelegateActions } = await wallet.signDelegateActions({
  delegateActions: [
    {
      receiverId: "contract.near",
      actions: [{
        type: "FunctionCall",
        params: {
          methodName: "method_name",
          args: {},
          gas: "30000000000000",
          deposit: "0",
        },
      }],
    },
  ],
});

// Send to relayer for execution
await fetch("/api/relay", {
  method: "POST",
  body: JSON.stringify({ signedDelegateActions }),
});
```

<Tip>
Delegate actions allow you to pay for gas fees on behalf of your users, improving their experience.
</Tip>

## Complete Example

<CodeGroup>
```tsx React NFT Minting
import { FC, useState } from "react";
import { NearWalletBase } from "@hot-labs/near-connect";
import { FinalExecutionOutcome } from "@near-js/types";

interface Props {
  wallet: NearWalletBase;
}

export const MintNFT: FC<Props> = ({ wallet }) => {
  const [loading, setLoading] = useState(false);
  const [result, setResult] = useState<FinalExecutionOutcome>();

  const handleMint = async () => {
    setLoading(true);
    try {
      const result = await wallet.signAndSendTransaction({
        receiverId: "nft.contract.near",
        actions: [
          {
            type: "FunctionCall",
            params: {
              methodName: "nft_mint",
              args: {
                token_id: `token-${Date.now()}`,
                metadata: {
                  title: "My NFT",
                  description: "An awesome NFT",
                  media: "https://example.com/nft.png",
                },
                receiver_id: (await wallet.getAccounts())[0].accountId,
              },
              gas: "30000000000000",
              deposit: "10000000000000000000000", // 0.01 NEAR
            },
          },
        ],
      });
      
      setResult(result);
      alert("NFT minted successfully!");
    } catch (error) {
      console.error("Failed to mint:", error);
      alert("Failed to mint NFT");
    } finally {
      setLoading(false);
    }
  };

  return (
    <div>
      <button onClick={handleMint} disabled={loading}>
        {loading ? "Minting..." : "Mint NFT"}
      </button>
      
      {result && (
        <div>
          <p>Transaction Hash: {result.transaction.hash}</p>
          <a
            href={`https://explorer.near.org/transactions/${result.transaction.hash}`}
            target="_blank"
            rel="noopener noreferrer"
          >
            View on Explorer
          </a>
        </div>
      )}
    </div>
  );
};
```

```typescript Token Swap
import { NearWalletBase } from "@hot-labs/near-connect";

export async function swapTokens(
  wallet: NearWalletBase,
  tokenIn: string,
  tokenOut: string,
  amountIn: string
) {
  const accounts = await wallet.getAccounts();
  const accountId = accounts[0].accountId;

  // Multi-step swap transaction
  const results = await wallet.signAndSendTransactions({
    transactions: [
      // 1. Register user if needed
      {
        receiverId: tokenOut,
        actions: [{
          type: "FunctionCall",
          params: {
            methodName: "storage_deposit",
            args: { account_id: accountId },
            gas: "30000000000000",
            deposit: "1250000000000000000000", // 0.00125 NEAR
          },
        }],
      },
      // 2. Swap tokens
      {
        receiverId: tokenIn,
        actions: [{
          type: "FunctionCall",
          params: {
            methodName: "ft_transfer_call",
            args: {
              receiver_id: "exchange.near",
              amount: amountIn,
              msg: JSON.stringify({
                token_out: tokenOut,
                min_amount_out: "0",
              }),
            },
            gas: "180000000000000",
            deposit: "1", // 1 yoctoNEAR
          },
        }],
      },
    ],
  });

  return results;
}
```

## Best Practices

<AccordionGroup>
  <Accordion title="Always specify gas amounts">
    Provide explicit gas amounts for function calls. The default may be insufficient for complex operations. 30 TGas (`30000000000000`) is a good starting point.
  </Accordion>

  <Accordion title="Use yoctoNEAR for deposits">
    All NEAR amounts must be specified in yoctoNEAR (10^-24 NEAR). Use `1` + 24 zeros for 1 NEAR, or use a conversion library.
  </Accordion>

  <Accordion title="Handle errors gracefully">
    Always wrap transaction calls in try-catch blocks and provide meaningful error messages to users.
  </Accordion>

  <Accordion title="Check return values">
    Parse and validate return values from smart contract calls. Don't assume success based on transaction completion.
  </Accordion>

  <Accordion title="Group related actions">
    When possible, include multiple related actions in a single transaction. This ensures atomicity and saves on fees.
  </Accordion>
</AccordionGroup>
