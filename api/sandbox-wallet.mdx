## Overview

The `SandboxWallet` class provides a secure way to interact with NEAR wallets by executing wallet code in a sandboxed iframe environment. This isolation ensures that wallet operations are performed safely without direct access to the main application context.

## Constructor

```typescript

```

<ParamField path="connector" type="NearConnector" required>
  The NEAR connector instance that manages the connection to the NEAR network.

<ParamField path="manifest" type="WalletManifest" required>

</ParamField>

## Usage Example

```typescript
import { SandboxWallet } from 'near-connect';

// Create a sandbox wallet instance
const wallet = new SandboxWallet(connector, walletManifest);

// Sign in
const accounts = await wallet.signIn({
  network: 'testnet'
});

// Send a transaction
const outcome = await wallet.signAndSendTransaction({
  receiverId: 'example.testnet',
  actions: [
    {
      type: 'FunctionCall',
      params: {
        methodName: 'myMethod',
        args: {},
        gas: '30000000000000',
        deposit: '0'
      }
    }
  ]
});
```

## Security

The `SandboxWallet` executes wallet code in an isolated iframe environment, providing an additional layer of security by:

- Preventing direct access to the main application's DOM and JavaScript context
- Isolating wallet permissions and capabilities
- Controlling communication through a message-passing interface

This approach is particularly useful for integrating third-party wallets while maintaining security boundaries.
