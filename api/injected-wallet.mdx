## Overview

The `InjectedWallet` class provides a wrapper around browser-injected NEAR wallet extensions. It acts as a bridge between the NEAR Connector and wallet extensions that inject themselves into the browser's global context (such as browser extension wallets).

## Constructor

```typescript
constructor(connector: NearConnector, wallet: NearWalletBase)
```

<ParamField path="connector" type="NearConnector" required>
  The NEAR connector instance that manages the connection to the NEAR network.
</ParamField>

<ParamField path="wallet" type="NearWalletBase" required>
  The injected wallet instance from the browser extension.
</ParamField>

## Properties

<ResponseField name="connector" type="NearConnector">
  Reference to the NEAR connector instance.
</ResponseField>

<ResponseField name="wallet" type="NearWalletBase">
  Reference to the underlying injected wallet implementation.
</ResponseField>

<ResponseField name="manifest" type="WalletManifest">
  The wallet manifest from the injected wallet, accessed via a getter.
</ResponseField>

## Usage Example

```typescript
import { InjectedWallet } from 'near-connect';

// Assuming a wallet was injected by a browser extension
const injectedWalletBase = window.nearWallets?.['my-wallet'];

if (injectedWalletBase) {
  const wallet = new InjectedWallet(connector, injectedWalletBase);
  
  // Sign in
  const accounts = await wallet.signIn({
    network: 'mainnet'
  });
  
  console.log('Wallet manifest:', wallet.manifest);
  
  // Send a transaction
  const outcome = await wallet.signAndSendTransaction({
    receiverId: 'example.near',
    actions: [
      {
        type: 'Transfer',
        params: {
          deposit: '1000000000000000000000000' // 1 NEAR
        }
      }
    ]
  });
}
```

## Error Handling

<Warning>
The `signAndSendTransaction` and `signAndSendTransactions` methods will throw an error with the message "No result from wallet" if the injected wallet returns an empty result.
</Warning>

```typescript
try {
  const outcome = await wallet.signAndSendTransaction(params);
} catch (error) {
  if (error.message === 'No result from wallet') {
    console.error('Wallet failed to return a transaction result');
  }
}
```

## Compatibility

The `InjectedWallet` class includes compatibility handling for different wallet implementations that may return results in varying formats. This ensures consistent behavior across different browser extension wallets.
