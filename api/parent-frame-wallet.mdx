## Overview

The `ParentFrameWallet` class enables wallet functionality for applications running inside an iframe by communicating with the parent frame via the `postMessage` API. This is useful for embedded applications that need to delegate wallet operations to their parent context.

```typescript

```

  The NEAR connector instance that manages the connection to the NEAR network.
</ParamField>

<ParamField path="manifest" type="WalletManifest" required>

</ParamField>

## Core Method

### callParentFrame

The internal method that handles all communication with the parent frame.

```typescript
callParentFrame(method: string, params: any): Promise<any>
```

<ParamField path="method" type="string" required>
  The method name to call on the parent frame (e.g., "near:signIn", "near:signAndSendTransaction").
</ParamField>

<ParamField path="params" type="any" required>
  Parameters to pass to the parent frame method.
</ParamField>

**Returns:** Promise that resolves with the result from the parent frame or rejects with an error.

<Info>
This method uses the `postMessage` API with message type "near-wallet-injected-request" and listens for "near-wallet-injected-response" messages. Each request is assigned a unique ID using UUID v4 for matching responses.
</Info>

## Usage Example

```typescript
import { ParentFrameWallet } from 'near-connect';

// In an iframe-embedded application
const wallet = new ParentFrameWallet(connector, walletManifest);

// Sign in via parent frame
const accounts = await wallet.signIn({
  network: 'testnet'
});

// Send a transaction through parent frame
const outcome = await wallet.signAndSendTransaction({
  receiverId: 'example.testnet',
  actions: [
    {
      type: 'FunctionCall',
      params: {
        methodName: 'myMethod',
        args: { value: 42 },
        gas: '30000000000000',
        deposit: '0'
      }
    }
  ]
});
```

## Parent Frame Implementation

The parent frame must implement a message listener to handle requests from the embedded application:

```javascript
window.addEventListener('message', async (event) => {
  if (event.data.type === 'near-wallet-injected-request') {
    const { id, method, params } = event.data;
    
    try {
      let result;
      
      switch (method) {
        case 'near:signIn':
          result = await handleSignIn(params);
          break;
        case 'near:signAndSendTransaction':
          result = await handleTransaction(params);
          break;
        // ... handle other methods
      }
      
      // Send success response
      event.source.postMessage({
        type: 'near-wallet-injected-response',
        id,
        success: true,
        result
      }, event.origin);
    } catch (error) {
      // Send error response
      event.source.postMessage({
        type: 'near-wallet-injected-response',
        id,
        success: false,
        error: error.message
      }, event.origin);
    }
  }
});
```

## Security Considerations

<Warning>
The `callParentFrame` method uses `postMessage` with `"*"` as the target origin, which allows communication with any parent frame. In production, you should validate the parent frame's origin and use a specific origin instead of `"*"`.
</Warning>

<Tip>
Always verify that your application is running in an iframe before using `ParentFrameWallet` to avoid errors when trying to communicate with a non-existent parent frame.
</Tip>

```typescript
if (window.self !== window.top) {
  // Safe to use ParentFrameWallet
  const wallet = new ParentFrameWallet(connector, manifest);
} else {
  // Use a different wallet implementation
}
```

## Message Protocol

The communication protocol uses the following message format:

**Request (iframe → parent):**
```json
{
  "type": "near-wallet-injected-request",
  "id": "unique-uuid-v4",
  "method": "near:signIn",
  "params": { /* method parameters */ }
}
```

**Response (parent → iframe):**
```json
{
  "type": "near-wallet-injected-response",
  "id": "unique-uuid-v4",
  "success": true,
  "result": { /* method result */ }
}
```

**Error Response:**
```json
{
  "type": "near-wallet-injected-response",
  "id": "unique-uuid-v4",
  "success": false,
  "error": "Error message"
}
```
